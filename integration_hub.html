The Architecture

The information hub architecture centralizes metadata, and decentralizes data processing and storage.

User Zone

Users connect to the information hub to build and run jobs, and connect directly with remote agents to access data. Users utilize a central metadata hub through the internet, whilst safely restricting data to internal networks.

Metadata Zone

The Information Hub stores metadata, such as table structures, and processing rules centrally.

Data Zone

Remote agents interact with data sources to perform data transformation and processing. Remote agents and data sources can be scaled indefinitely for small scale and very large implementations.

Secure or Share

For sensitive data, remote agents can be data locked, ensuring data never leaves the data zone. For sharing, remote agents can share data with other remote agents or users as needed.
